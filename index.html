3.正弦函数和余弦函数的性质

1.周期性

可以发现，正弦函数和余弦函数的图象都是波浪起伏，连绵不断，好像是其中的某一段复制了许多份。

像这样，我们称正弦函数和余弦函数具有周期性。

一般地，设函数 f(x)f(x)f(x) 的定义域为 DDD，如果存在一个非零常数 TTT，使得对每一个 x∈Dx\in Dx∈D 都有 x+T∈Dx+T\in Dx+T∈D，并且

f(x+T)=f(x)f(x+T)=f(x) f(x+T)=f(x)

那么函数 f(x)f(x)f(x) 就称为__周期函数__，TTT 称为这个函数的__周期__。

周期函数的周期不止一个，例如，如果 TTT 是周期函数 f(x)f(x)f(x) 的周期，那么 2T2T2T，3T3T3T，……也一定是周期函数 f(x)f(x)f(x) 的周期。

在周期函数 f(x)f(x)f(x) 的所有周期中，必然有一些正数，在这些正数中最小的那个，叫做函数 f(x)f(x)f(x) 的最小正周期。

根据诱导公式，我们有：

sin⁡(α+2π)=sin⁡αcos⁡(α+2π)=cos⁡α\sin(\alpha+2\pi)=\sin\alpha \\ \cos(\alpha+2\pi)=\cos\alpha sin(α+2π)=sinαcos(α+2π)=cosα

所以，

正弦函数是周期函数，它的最小正周期是 2π2\pi2π；余弦函数是周期函数，它的最小正周期是 2π2\pi2π。

当然，要研究的周期不止这么简单，现在来研究函数 y1=ksin⁡(ωx+ϕ)y_1=k\sin(\omega x+\phi)y1​=ksin(ωx+ϕ) 和函数 y2=kcos⁡(ωx+ϕ)y_2=k\cos(\omega x+\phi)y2​=kcos(ωx+ϕ) 的周期性：

在这两个表达式中，kkk 不会对函数的周期产生任何影响，可以忽略。

令 z=ωx+ϕz=\omega x+\phiz=ωx+ϕ，则

y1=ksin⁡zy2=kcos⁡zy_1=k\sin z \\ y_2=k\cos z y1​=ksinzy2​=kcosz

由诱导公式

sin⁡(α+2π)=sin⁡αcos⁡(α+2π)=cos⁡α\sin(\alpha+2\pi)=\sin\alpha \\ \cos(\alpha+2\pi)=\cos\alpha sin(α+2π)=sinαcos(α+2π)=cosα

得到，

sin⁡(z+2π)=sin⁡αcos⁡(z+2π)=cos⁡α\sin(z+2\pi)=\sin\alpha \\ \cos(z+2\pi)=\cos\alpha sin(z+2π)=sinαcos(z+2π)=cosα

Start keeping good coding habits from A+B Problem
2023/10/22

Now, we believe every OIer can write a code of A+B problem very easily.

#include<bits/stdc++.h>
using namespace std;
int main(){
    int a,b;
    cin>>a>>b;
    cout<<a+b;
}

So easy,isn't it? But if I say there are many problems in this code, can you believe?

First,do you truly end this program? Where is your return 0?

#include<bits/stdc++.h>
using namespace std;
int main(){
    int a,b;
    cin>>a>>b;
    cout<<a+b;
    return 0;
}

Second,do you create a variable to store the answer?

Maybe you think it is not necessary. But it is. As you see, a and b are both integers. If the value of a+b is greater than 2147483647,or less than -2147483648,the output will be wrong. So we can create a variable to store the answer.

#include<bits/stdc++.h>
using namespace std;
int main(){
    int a,b;
    cin>>a>>b;
    long long answer=(long long)(a+b);
    cout<<answer;
    return 0;
}

Third,do you think the input and the output is too slow?If so,we can change our way to input or output.

#include<bits/stdc++.h>
using namespace std;
int main(){
    int a,b;
    scanf("%d%d",&a,&b);
    long long answer=(long long)(a+b);
    printf("%lld",answer);
    return 0;
}

or

#include<bits/stdc++.h>
using namespace std;
int main(){
    std::ios::sync_with_stdio(false);
    int a,b;
    cin>>a>>b;
    long long answer=(long long)(a+b);
    cout<<answer;
    return 0;
}

They are the optimizations of A+B problem. But there are many other optimization ways can't show out at this problem. I will tell you some of them next time.Bye!
